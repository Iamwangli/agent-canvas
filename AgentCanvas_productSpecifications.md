
# 规格书

## AgentCanvas 最终产品规格（实现版）

**文档更新日期**：2026-04-20  
**版本**：2.1（新增实现状态标记）

### 一、产品定位

无限画布，用户通过树形结构的对话节点与 Agent 根节点，构建多智能体协作的推理链。每个节点代表一轮完整的单次问答（用户→AI 或 AI→AI）。上下文沿父链向上游追溯，隐藏节点可选择性排除自身内容，但不影响子节点继续向上索引。**支持多会话管理**（文件模型），所有画布数据自动持久化到浏览器本地存储。

---

### 二、节点类型与视觉规范

| 节点类型 | 形状 | 样式 | 内容结构 | 其他 | 状态 |
|---------|------|------|----------|------|------|
| **Agent 根节点** | 椭圆 | 浅蓝色填充，黑色边框，文字加粗 | 显示 Agent 名称（可编辑）、模型标识（如 DeepSeek, Qwen） | 不可删除 | ✅ |
| **对话节点（普通）** | 圆角矩形（rx=8） | 白色填充，灰色实线边框 | **固定上下两部分**：上半区问题（用户输入），下半区回复（AI 生成）。**不可编辑** | 宽度 280px，高度自适应 | ✅ |
| **对话节点（AI 自动创建）** | 同上 | 金色虚线边框 + 淡黄色背景，持续呼吸动画 3 次 | 同上 | 高亮提醒用户 | ✅ |
| **隐藏节点** | 同上 | 灰色虚线边框，半透明填充（opacity 0.4），内容可见 | 同上 | 画布上可见，但 AI 索引时忽略其自身内容 | ✅ |

- **连线**：有向贝塞尔曲线（**子节点 → 父节点**），颜色与父节点所属 Agent 根节点色调对应（可选），末端带箭头。**每个节点拥有四个方向的连接点（上下左右）**，用户可从任意方向拖拽连线。✅
- **尺寸自适应**：根据问题/回复的文本量自动增长高度，宽度固定 280px。支持换行、Markdown 基本格式。✅

### 三、核心交互操作

#### 3.1 初始化画布与多会话管理

- **会话（文件）模型**：每个会话是一个独立的画布状态，包含根节点和所有对话节点。所有会话保存在 `files` 数组中，当前激活的会话由 `currentFileId` 标识。✅
- **自动持久化**：所有会话数据自动保存到 `localStorage`（键名 `agent-canvas-files`），页面刷新后自动恢复。✅
- **初始化**：首次打开时自动创建一个“未命名会话”，并根据后端 Agent 列表生成根节点。✅
- **历史侧边栏**：画布左侧显示所有历史会话列表，每个会话显示名称和创建时间，支持**恢复**和**删除**。删除会话时至少保留一个。✅

#### 3.2 创建对话节点

- **手动创建（双击节点）**：双击任意节点（Agent 根节点或对话节点），会在该节点的**下方**创建一个新的空白对话节点，并自动建立连线（子节点 → 父节点）。新节点进入输入模式，输入框聚焦。✅
- **右键创建子节点（支持方向）**：右键点击对话节点，菜单中出现“创建子节点”二级菜单，可选择**上方、右方、下方、左方**四个方向。新节点出现在指定方向，自动建立精准连线（子节点面向父节点一侧的 Handle 连接至父节点对应侧）。**只有已回复的对话节点或根节点允许创建子节点**（未回复的节点会提示“不能创建子节点”）。✅
- **输入与发送**：在输入框中输入问题，按回车或点击“发送”按钮。系统收集上下文（沿父链向上，跳过隐藏节点），调用对应 Agent API，将回复显示在下半区。发送后节点变为只读，不可编辑。✅

#### 3.3 文件上传

- **对话节点输入区**增加“上传 TXT”按钮。点击后选择本地 `.txt` 文件，读取文件内容（UTF-8 编码）。✅
- **交互优化**：上传后不填充输入框，而是在输入框上方显示文件标签（📄 文件名 + 移除按钮）。✅
- **发送行为**：发送时后端接收到的 `question` 参数会拼接文件内容（格式：`[上传文件：xxx.txt]\n内容：\n...`），但**节点上保存的 `question` 字段仅包含用户输入的纯文本问题**，保持界面整洁。✅

#### 3.4 AI 自动创建节点

- **开关**：工具栏提供“🤖 AI自动节点”开关，默认开启，状态持久化。✅
- **触发条件**：当前 Agent 回复内容中表达需要向另一个 Agent 请求信息（例如“我需要向 Qwen 查询数据”），或基于预设关键词规则。✅
- **行为**：在当前对话节点的父节点下创建新节点，自动填入问题并自动发送（调用目标 Agent API）。新节点使用高亮样式（金色虚线边框 + 淡黄色背景 + 呼吸动画）。✅
- **防滥用**：同一父节点下连续自动创建超过 3 次时弹出确认框：“AI 想要继续创建节点，是否允许？”✅

#### 3.5 节点操作（手动）

| 操作 | 方法 | 行为 | 状态 |
|------|------|------|------|
| **移动节点** | 拖拽节点主体 | 所有连线自动重绘 | ✅ |
| **重新指定父节点** | 拖拽节点**入线**（来自父节点的连线）末端到另一个节点 | 断开原父节点，连接新父节点。需检查循环：新父节点不能是该节点的后代。**连线方向限制**：只能从子节点拖向父节点，反向拖拽会被拒绝并提示。 | ✅ |
| **删除节点** | 右键菜单 → “删除” | 弹出确认框：“删除节点后，其子节点将自动连接到上级节点。确定删除吗？”确认后：删除该节点及其所有连线；子节点自动连接到被删除节点的原父节点（若原父节点不存在则变为孤立节点）。 | ✅ |
| **隐藏节点** | 右键菜单 → “隐藏” | 节点变为灰色虚线边框、半透明填充。该节点自身在 AI 索引上下文时**完全忽略**，但其子节点仍然可见，且向上追溯时会跳过该隐藏节点，继续访问更上层的祖先。 | ✅ |
| **取消隐藏** | 右键菜单 → “取消隐藏” | 恢复原样式，重新纳入上下文索引。 | ✅ |
| **复制节点** | 右键菜单 → “复制节点” | 生成一个新的自由节点（无父节点），复制原节点的问题和回复内容。新节点位置为鼠标点击处。 | ✅ |
| **修正问答** | 单击父节点，创建新节点 | 手动建立新分支。原节点及其子节点保留，用户可通过“重新指定父节点”将原节点的子节点重连到新节点。 | ✅ |

> **右键菜单优化**：根据节点当前隐藏状态，动态显示“隐藏节点”或“取消隐藏”，二者不会同时出现。✅

#### 3.6 连线操作

- **手动连线**：从子节点的任意方向连接点（Handle）拖拽到父节点的任意方向连接点，即可建立连线，同时更新子节点的 `parentId`。**连线方向必须是从子节点指向父节点**（反向拖拽会被拒绝并提示）。✅
- **删除连线**：双击连线即可删除，子节点变为自由节点（`parentId = null`），其所有后代节点仍保持原有关系。✅
- **连线稳定性**：手动连线的 Handle 信息会被保留，画布刷新或节点移动后连线不会自动吸附到顶部。✅

---

### 四、上下文索引规则（树形结构，单父节点）

- **定义**：对于任意对话节点 N，其上下文 = 从 N 开始，沿父链向上追溯所有祖先节点（直到 Agent 根节点），收集每个节点的**问题+回复**内容。✅
- **隐藏节点处理**：若祖先节点中遇到隐藏节点，**跳过该节点自身的内容**，继续向上追溯（即不阻断子孙节点对更上层祖先的访问）。✅
- **排除**：非祖先节点的任何其他节点（兄弟、旁支）不参与上下文。✅
- **孤立节点**：没有父节点的对话节点无法发送（因为没有 Agent 归属）。用户必须手动将其连接到某个 Agent 根节点或其他节点。✅

---

### 五、Agent 路由与 API 调用

- 每个对话节点在发送时，沿父链向上找到**最近的 Agent 根节点**（若找不到，则无法发送，提示用户先连接到根节点）。✅
- 调用对应大模型 API（如 DeepSeek、Qwen），携带上下文（按顺序拼接的问答历史）。✅
- Agent 根节点可配置 API Key、端点、模型名称等参数（通过双击根节点进行配置，UI 待实现）。⚠️ 待开发

---

### 六、禁止多父节点 & 循环检测

- 每个对话节点**最多只有一个父节点**。UI 上，当节点已有父节点时，再次尝试连接新父节点会先断开原连接（即重新指定父节点操作）。✅
- 禁止形成循环：如果尝试将某个节点的父节点设置为其后代节点，系统拒绝并提示“不能创建循环引用”。✅

---

### 七、删除节点时的子节点重连规则（详细）

删除节点 D 后：
- 获取 D 的直接子节点集合 C。
- 获取 D 的原父节点 P（如果存在）。
- 对于每个 c ∈ C：
  - 如果 P 存在，则将 c 的父节点设置为 P（即 c 连接到 P）。
  - 如果 P 不存在（D 是 Agent 根节点的直接子节点），则 c 变为孤立节点（parentId = null）。✅
- 删除 D 及其所有连线。

> 弹出确认框提示：“删除节点后，其子节点将自动连接到上级节点。确定删除吗？”✅

---

### 八、AI 自动节点开关的 UI 与行为

- **开关位置**：画布工具栏，按钮式开关，显示“🤖 自动节点”。✅
- **默认状态**：开启。✅
- **开启时**：AI 在回复中若触发条件，会自动创建高亮节点。✅
- **关闭时**：AI 不会创建任何新节点，仅进行回复。✅
- 自动创建的节点会记录标记 `isAutoCreated: true`，用户可手动删除或重新连线。✅

---

### 九、多会话管理（新对话与历史记录）

- **“新对话”按钮**：点击工具栏的“+ 新对话”按钮，会**将当前画布保存到历史列表**（如果当前画布包含至少一个对话节点），然后**清空所有对话节点，只保留 Agent 根节点**，创建一个全新的空白画布。✅
- **历史侧边栏**：左侧显示所有历史会话，每个会话条目包含名称和创建时间。点击“恢复”可切换回对应画布；点击“删除”可删除该会话（至少保留一个会话）。✅
- **自动保存**：每次对画布进行修改，当前会话的节点状态都会自动保存到 `files` 数组中，并持久化到 `localStorage`。✅

### 十、技术实现要点

- **数据结构**（保持不变，已实现）✅
- **上下文收集函数**（`collectContext`，已实现）✅
- **渲染库**：React Flow 11.10.4+，自定义节点包含四个方向的 Handle，连线类型为 `smoothstep`。✅
- **API 调用**：后端 Express 服务，根据 `agentId` 路由到不同模型。✅
- **持久化**：Zustand + `persist` 中间件，`localStorage` 键名 `agent-canvas-files`。✅

### 十一、边界情况与约束

| 场景 | 处理方式 | 状态 |
|------|----------|------|
| 对话节点没有父节点（孤立） | 禁止发送，提示“该节点未连接到任何 Agent，请先连接至 Agent 根节点” | ✅ |
| 删除 Agent 根节点 | **禁止删除**，右键菜单中无删除选项 | ✅ |
| 隐藏节点作为唯一父节点 | 子节点正常可见，向上追溯时跳过隐藏节点，继续访问更上层祖先 | ✅ |
| 自动创建节点过于频繁 | 同一父节点下连续自动创建超过 3 次时弹出确认框 | ✅ |
| 超大画布性能 | 限制单画布节点数上限（如 500 个），超出时提示（待完善） | ⚠️ |
| 文件上传 | 仅支持 `.txt` 文件，编码 UTF-8 | ✅ |
| 连线方向错误 | 提示“连线方向错误：请从子节点拖向父节点” | ✅ |
| 右键菜单互斥 | 根据隐藏状态只显示“隐藏节点”或“取消隐藏” | ✅ |
| 方向创建节点连线精准 | 子节点与父节点通过对应侧 Handle 连接 | ✅ |
